# PyGit — мини-реализация Git на Python

## Описание проекта

**PyGit** — это учебная реализация упрощённой системы контроля версий, вдохновлённая архитектурой Git.

Проект реализует базовые механизмы Git:

* хранение объектов через SHA-1;
* blob-объекты для хранения содержимого файлов;
* tree-объекты для представления структуры каталогов;
* commit-объекты для хранения истории изменений;
* индекс для отслеживания добавленных файлов;
* систему ссылок HEAD и веток;
* просмотр истории коммитов.

Цель проекта — разобраться во внутреннем устройстве Git и реализовать его основные концепции самостоятельно.

## Возможности

Реализованы следующие команды:

| Команда                     | Описание                           |
| --------------------------- | ---------------------------------- |
| `pygit init`                | Создание нового репозитория        |

| `pygit commit -m "message"` | Создание нового коммита            |

## Как работает PyGit

### Объекты

Как и настоящий Git, PyGit хранит данные в виде объектов.

Каждый объект:

1. получает заголовок с типом и размером;
2. объединяется с содержимым;
3. хешируется через SHA-1;
4. сжимается через zlib;
5. сохраняется в `.pygit/objects`.

Формат хранения:

```
.py.git/objects/ab/cdef123456789...
```

где:

* первые 2 символа SHA-1 — имя директории;
* остальные символы — имя файла.

---

## Типы объектов

### Blob

Хранит содержимое файла.

Пример:

```
file.txt
    |
    v
 Blob
    |
    v
 SHA-1
```

---

### Tree

Описывает структуру директорий.

Хранит записи:

```
(mode, name, sha)
```

Пример:

```
src/
 ├── main.py
 └── utils.py
```

представляется как дерево:

```
Tree
 |
 ├── src -> Tree
 |
 └── README.md -> Blob
```

---

### Commit

Хранит информацию о версии проекта:

```
commit
 |
 ├── tree
 |
 ├── parent
 |
 ├── author
 |
 └── message
```

Каждый новый commit содержит ссылку на предыдущий, благодаря чему строится история.

## Использование

### 1. Инициализация репозитория

```bash
python -m pygit init
```

Создастся структура:

```
.pygit/
├── objects/
├── refs/
│   └── heads/
└── HEAD
```

---

### 2. Добавление файла

Создадим файл:

```bash
echo "Hello PyGit" > test.txt
```

Добавляем его:

```bash
python -m pygit add test.txt
```

После этого файл попадёт в индекс.

---

### 3. Создание дерева

```bash
python -m pygit write-tree
```

Команда создаёт tree-объект и выводит его SHA-1:

```
8f3a12b45c...
```

---

### 4. Создание коммита

```bash
python -m pygit commit -m "Initial commit"
```

Результат:

```
[4fa92bd] Initial commit
```

---

### 5. Просмотр истории

```bash
python -m pygit log
```

Пример вывода:

```
commit 4fa92bd34d...

Author: user

Initial commit


commit 98ab12345...

Author: user

Added new file
```

---

## Отличия от настоящего Git

PyGit является учебной реализацией и не поддерживает:

* ветки в полном объёме;
* merge;
* checkout;
* удаление файлов;
* восстановление файлов;
* удалённые репозитории;
* SSH/HTTP протоколы;
* хук-систему Git.
